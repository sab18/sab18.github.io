## sab18.github.io

### 1. Updating Website Layout (Format/Structure)
1. Edit the relevant Python script(s) or template logic:
   - `update_site.py`: Controls the layout and structure for homepage, IRL Projects, Digital Projects, and About pages.
   - `update_project_page.py`: Controls the structure for individual project pages using YAML content files.
   - `style.css`: Controls all site-wide styling (colors, margins, fonts, etc.).
2. To update the layout of:
   - **Homepage**: Run `update_site.py` after making changes to scripts or `style.css`.
   - **IRL Projects/Digital Projects/About**: Run `update_site.py` after making changes.
   - **Project Pages**: Run `update_project_page.py` to update the format/layout of all project pages using the YAML content files in `project_content/`.
3. All pages (except project pages) are auto-regenerated by running `update_site.py`.
4. For project pages, run `update_project_page.py` to apply new layout changes to all project pages without overwriting their content.

### 2. Changing Page Contents (Adding/Editing Projects)
1. To add a new project:
   - Use `generate_page.py` to create a new project. This will:
     - Create a YAML content file for the project in `project_content/`.
     - Add the project to the correct list in `projects_list.py`.
     - Copy the default header image if needed.
   - Then run `update_project_page.py` to generate the project HTML file from the YAML content.
2. To update project lists (spotlight, IRL, digital):
   - Edit `projects_list.py` to add/remove/reorder projects in the appropriate lists.
   - Run `update_site.py` to regenerate all index/listing pages with the new content.
3. To update About, Homepage, IRL Projects, or Digital Projects content:
   - Edit the relevant content in the Python scripts or the HTML templates.
   - Run `update_site.py` to apply changes.
4. To update the content of a project page:
   - Edit the corresponding YAML file in `project_content/`.
   - Run `update_project_page.py` to regenerate the project HTML file with the updated content.

### General Maintenance Notes
- All layout and style changes should be made in `update_site.py`, `update_project_page.py`, or `style.css` for consistency.
- The menu bar order and labels are controlled by `menu_list` in `projects_list.py` and the label logic in `update_site.py`.
- All images should be placed in the appropriate folder under `assets/images/`.
- The logo should be stored as `assets/images/logo.png` for consistency.
- To ensure all pages are up to date, always run `update_site.py` after making changes to layout, style, or project lists, and `update_project_page.py` after changing project page layout or content.
- For new project pages, always use `generate_page.py` to ensure correct structure and registration in the project lists, then run `update_project_page.py`.
- If you change the structure of the project lists (e.g., add new fields), update both `update_site.py` and `generate_page.py` accordingly.
